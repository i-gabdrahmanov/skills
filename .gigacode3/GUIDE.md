# Руководство: как пользоваться обвязкой (GigaCode Forge)

> **Кому:** тому, кто **запускает** фичи через обвязку. Практический how-to: установить один раз,
> запустить фичу, понять что спросят на гейтах, резюмировать, чистить, настраивать, чинить.
> Хочешь «почему так устроено» → [`FORGE.md`](FORGE.md). Разработчику обвязки →
> [`AGENT-RUNBOOK.md`](AGENT-RUNBOOK.md). Деплой хуков детально → [`hooks/DEPLOY.md`](hooks/DEPLOY.md).

---

## 1. Что это за пять минут

Обвязка проводит фичу Java/Spring **от заявки до pull request** одним управляемым конвейером:
бизнес-требования → проектирование → задачи в Jira → код с тестами → проверка качества → спецификация
→ stacked-PR. Ключевое: **правила качества форсит сам рантайм** (хуки), а не «добрая воля» модели —
пропустить тесты, выкатить без проверок или сделать рискованное «молча» нельзя.

---

## 2. Установка (один раз на машине)

Получи папку обвязки любым способом (git clone / распакованный архив / общий каталог) и запусти
из неё установщик — он ставит **всё сразу** и сам проверяет результат:

```bash
bash install.sh                      # ставит в ~/.gigacode (прод). Для теста: bash install.sh ~/.qwen
```
`install.sh` проверит пред-условия (python3, CLI), скопирует hooks+skills+доки в дом рантайма,
вольёт блок hooks в settings, прогонит `doctor` и напечатает «что дальше». Обновиться позже —
повторно запустить `install.sh`.

⚠️ **Запускай рантайм ВСЕГДА с флагом хуков** — без него контроль качества выключен (хуки не
регистрируются):
```bash
gigacode --experimental-hooks -p "<задача>"        # или интерактивно: gigacode --experimental-hooks
```

Перед каждым серьёзным прогоном — быстрый self-check, что контроль реально включён:
```bash
python3 ~/.gigacode/hooks/preflight.py --project .
```
`✅` — можно работать. `❌ ENFORCEMENT OFF` — подними флаг/деплой, не работай вслепую.

## 3. Быстрый старт — запустить фичу

В сессии рантайма (запущенной с `--experimental-hooks`), из корня репозитория кода:

```
/skills feature-pipeline KIDPPRB-8639
```
или просто словами: **«проведи фичу по задаче KIDPPRB-8639»** / «сделай фичу: автозакрытие пустых заявок».

- Вход — **ключ Jira** (`[A-Z]+-\d+`) или свободная идея.
- Дальше обвязка ведёт тебя по фазам и останавливается на **гейтах** — точках, где нужно твоё «да».

> Первый запуск на новом проекте: обвязка сама создаст `ground/pipeline.json` и спросит то, что не смогла
> определить (ключ Jira-проекта, Bitbucket, нужен ли `git init`). См. §9.

## 4. Что произойдёт по фазам и что у тебя спросят

| Фаза | Что делает система | Что нужно от тебя |
|---|---|---|
| **0 · BRD** | Собирает бизнес-требования (интервью или из идеи/Jira). Пишет на языке бизнеса. | **Гейт 1:** подтвердить BRD / отправить на доработку. |
| **Критичность** | Спрашивает важность фичи. | Выбрать **low / medium / high** (см. §6). Без выбора рискованные шаги заблокированы. |
| **1 · Grounding** | Берёт актуальный срез системы (модули, сущности, API). Не пересканирует, если обзор уже есть. | Обычно ничего. |
| **2 · Tech-Design** | Проектирует по grounding (не по коду): план + строгий SDD + разбивка на задачи. | **Гейт 2:** «делаем так / правки?». |
| **2.5 · Jira** | Готовит черновик Story + Sub-task. | **Гейт 3:** «создаём задачи?» (можно без Jira). |
| **3 · Build (TDD)** | По каждой задаче: **сначала тесты (они падают), потом код, пока не позеленеют**. | Обычно ничего; смотришь diff. |
| **4 · Verify** | Полный прогон тестов + покрытие. | Ничего (добор покрытия — автоматом). |
| **5 · Document** | Обновляет спецификацию; подмешивает изменения фичи в обзор системы. | Ничего. |
| **6 · Deliver** | Ветки + коммиты + stacked-PR + отчёт в Jira. | **Гейты 4–6:** «коммитим? пушим и создаём PR? отправляем отчёт?» |

На каждой фазе есть и **автоматическая проверка** (тесты компилируются и падают перед кодом, покрытие
добито, артефакты на месте, evidence собран) — пока она не зелёная, конвейер дальше не идёт.

---

## 5. Гейты: что это и как отвечать

- **Гейт = пауза перед необратимым/важным.** Система показывает, что собирается сделать, и ждёт «да».
- Отвечай по существу: «да / делаем», «правки: …», «нет». На «правки» система вернётся и переделает.
- **Необратимое без «да» не происходит** — создание задач, коммит, push, PR, отчёт в Jira. Это гарантия,
  а не формальность: даже если попросить «просто закоммить», без прохождения проверок рантайм заблокирует.

---

## 6. Критичность фичи (важно)

Сразу после BRD система спрашивает уровень важности — он задаёт, насколько строги проверки:

| Уровень | Когда | Эффект |
|---|---|---|
| **low** | эксперимент, non-prod, внутр. инструмент | почти всё проходит само; гейтятся доставка и опасные пути |
| **medium** | обычная прод-фича (по умолчанию) | код — через тесты и ревью; коммит/пуш/Jira под гейтами |
| **high** | платежи, персональные данные, доступы, инфраструктура | почти каждое действие требует подтверждения/доказательств |

Без выбора критичности рискованные действия **заблокированы** — это сделано намеренно.

---

## 7. Прерывания, резюм и несколько фич

Состояние пайплайна сохраняется **по фиче**, поэтому обрыв не теряет прогресс и фичи не мешают друг другу.

```bash
# какие фичи сейчас в работе / завершены:
python3 ~/.gigacode/skills/pipeline-state/scripts/read.py --skill feature-pipeline --list
```
- Прервался прогон → запусти снова: обвязка увидит незавершённую фичу и предложит **резюмировать** с того
  же места, начать заново или показать собранное.
- Можно вести несколько фич параллельно — у каждой свой стейт (`ground/statements/feature-pipeline/<feature>/`).

## 8. Логи и наблюдаемость

```bash
# живой лог текущего прогона (в отдельном терминале):
bash ~/.gigacode/hooks/watch-agents.sh

# сводные метрики доверия по всем прогонам (частота блокировок, сбоев, субагенты):
python3 ~/.gigacode/hooks/agentops.py --archive ~/.gigacode/ai-logs-archive
```
Всё поведение агента и субагентов пишется в `ground/ai-logs/<feature>/` (по прогону) и в общий архив
`~/.gigacode/ai-logs-archive/` (для анализа).

## 9. Конфигурация проекта — `ground/pipeline.json`

Создаётся автоматически при первом запуске; правится под проект. Ключевые поля:

| Поле | Зачем |
|---|---|
| `quality.coverage_threshold` | минимальное покрытие (дефолт 0.80) |
| `quality.tdd` | TDD по умолчанию `true` (тесты вперёд) |
| `quality.test_layer` | `service-unit` (моки; `@DataJpaTest` запрещён) или `mixed` (разрешить интеграционные) |
| `quality.token_budget` | бюджет на прогон (warn 80% / stop 120%) |
| `evidence.threshold` | полнота доказательного пакета перед доставкой (дефолт 0.95) |
| `autonomy.criticality` / `auto_max_risk` | критичность фичи и порог авто-прохода (см. §6) |
| `jira.*`, `bitbucket.*` | ключ проекта, workspace/repo (для фаз Jira и Deliver) |
| `conventions.migration_tool` | liquibase / flyway / none |

## 10. Уборка артефактов по задаче

Убрать локальные файлы пайплайна по Jira-ключу (BRD/design/SDD, стейт, логи, evidence фичи):

```
/skills cleanup-jira-artifacts KIDPPRB-8639
```
Сначала покажет **dry-run** (что удалит), спросит подтверждение, и только потом удалит. **Не трогает**
обзор системы (grounding), общий архив логов, git-ветки и сами задачи в Jira.

## 11. Траблшутинг (частые ситуации)

| Симптом | Причина | Что делать |
|---|---|---|
| Контроль не работает, ничего не блокируется | Рантайм запущен **без** `--experimental-hooks` (`0 hook entries`) | Перезапусти `gigacode --experimental-hooks`; прогони `preflight.py` |
| `preflight`/`doctor` ругается | Обвязка не развёрнута / skills не рядом с hooks | `bash deploy.sh ~/.gigacode` ещё раз |
| Гейт заблокировал commit/push | Не пройдены требования уровня (тесты, evidence, критичность) | Доведи шаг, на который указывает причина блокировки |
| «Не дают писать код» | TDD: для задачи ещё нет падающих тестов | Сначала тесты (фаза RED), потом код |
| Тест репозитория падает `initializationError` | `@DataJpaTest` в multimodule | Используй сервисный unit-тест с моками; либо `quality.test_layer=mixed` |
| Субагенты не запускаются | Рантайм не поднял subagent-инфру | Работа продолжится inline (деградация) — будет помечено; сообщи владельцу обвязки |
| Скилл не находится | Папка без валидного `SKILL.md` | `python3 hooks/validate_skills.py --skills ~/.gigacode/skills` |
